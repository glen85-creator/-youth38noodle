## 1. 배경 & 목표

### 배경
- 현재 랜딩은 초기 셋업 상태. 일부 사실 오류(브랜드 출범 연도), 내부 수치의 과도한 노출 등 이슈 존재.
- 브랜드 핵심 차별점인 **"100원 기부 선순환"**, **"매장 타입 다양성"**, **"지역연계형(다산38) 스토리"**가 미반영.
- 내일 실제 가맹 희망 고객 미팅이 예정되어 있어 **프린트 가능한 제안서 PDF** 필요.

### Phase E — PDF 제안서 기능 구현 (60~90분)
*(User 제안 2·3번)*

**기술 접근법**: 별도 라이브러리 없이 **브라우저 `window.print()` + 인쇄 최적화 CSS** 방식.  
이유:
- 추가 의존성 불필요, 즉시 작동
- 브라우저 네이티브 "PDF로 저장" 기능 활용 → 확정된 PDF 확보
- A4 레이아웃·페이지 분할 제어 가능
- 프린트 물리 출력도 동일 CSS로 커버

- [ ] E1. 상단 Nav에 **"📄 제안서 다운로드"** 버튼 추가
- [ ] E2. `/proposal` 라우트 또는 `?view=print` 쿼리로 제안서 전용 뷰 활성화
  - 랜딩과 동일한 섹션(Hero 요약/BrandStory/기부/메뉴/매장타입/가맹프로세스/Support/Contact 정보)
  - 애니메이션·모션 제거 (인쇄용)
  - 페이지별 자연스러운 분할 (`page-break-after`, `page-break-inside: avoid`)
- [ ] E3. 인쇄용 CSS (`@media print`)
  - A4 세로, 여백 20mm
  - 폰트 사이즈·색상 대비 조정 (잉크 절약)

- [ ] E4. 버튼 동작: 전용 뷰에 들어간 직후 `window.print()` 자동 호출 옵션 (또는 명시 버튼)
