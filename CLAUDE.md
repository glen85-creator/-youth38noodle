## 3. ⚠️ 수치 표기 원칙 (매우 중요)

**디테일한 금액·비율을 랜딩에 노출하면 법적·영업적 이슈 소지 → 유추 가능한 수준까지만**

### 허용 (유추 수준)
- 시그니처 메뉴 최저가: **3,800원**
- 메뉴 가격대: **3,800 ~ 8,800원**
- 예상 영업이익률: **20~30%** (매장 타입에 따라 상이)
- 총 초기 투자비용: **약 4,700만원~** (세부 내역 공개는 신중히)
- "본사 노마진 정책", "본사 보증금 없음"

### 금지 (디테일)
- 월 매출 구체액 예시 (~~"매출 2,000만원 기준 500만원 이익"~~)
- 원가·인건비·임차료 정확한 %/금액을 시각화한 수익구조
- 로열티/광고분담금 구체 요율 (`docs/franchise/` 가맹정의서로 재확인 필요)
- 표준손익계산서 상의 구체 수치 (내부 참고용이지 공개용 아님)

### 현재 코드에서 **수정 필요** (src/App.tsx)
- `Finance` 섹션의 "월 수익 구조 예시 (매출 2,000만원 기준)" 상세 브레이크다운 → **범위/개념 중심으로 추상화**
- `Finance` 초기 투자비용 상세 테이블 → 합계·범위 중심으로 완화 (또는 docs/franchise 근거로 재확정)
- `Metrics` 영업이익률 "30%" → "**20~30%**"
- FAQ의 "로열티 3% + 광고분담금 1% = 4%" → 가맹정의서 재확인 후 확정
- BrandStory 하단 **"1988년부터 변함없는 정성"** → 브랜드 출범은 2025년. 표현 재작성 필요 (예: "매일 아침 직접 우려내는 정성", "20년 노하우의 생면" 등)

## 4. 기술 스택

- React 19 / TypeScript 5.8 / Vite 6 / Tailwind **v4** (@theme 토큰) / motion (Framer Motion)

```bash
npm run dev      # vite --port=3000 --host=0.0.0.0
npm run build
npm run lint     # tsc --noEmit
npm run preview
```
